# Erfindung-Check Module: Detailed Description

## Purpose

The Erfindung-Check (Invention Check) module helps inventors and innovators assess whether their idea is novel and how it differs from existing solutions. It delivers a structured 8-step process that leads to one of three outcomes: **Neu wirkt plausibel** (novel), **Unklar** (unclear), or **Eher bekannt** (likely known), plus a concrete next action.

---

## Process Flow

The process guides users through eight steps:

| Step | Name | Purpose | Output |
|------|------|---------|--------|
| 0 | Einleitung | Explain what an invention is and why this check matters | User ready to proceed |
| 1 | Kategorie wählen | Choose category: Produkt/Mechanik, Material/Verfahren, Software, or Medizin | Category + search strategy |
| 2 | Markt scannen | Broad search (DE+EN) for similar products; build keyword set | 5–10 similar solutions + keyword list |
| 3 | Prior Art suchen | Patent and publication search (Espacenet, Google Patents, Scholar, PubMed, etc.) | Patent and publication tables |
| 4 | Treffer bewerten | Classify hits: same function/different mechanism, similar mechanism/different purpose, same function+mechanism | Evaluation table |
| 5 | Kern schärfen | Formulate core idea sentence, variants, keywords | Kernidee + variants + keywords |
| 6 | Ergebnis ableiten | Choose status and derive next action | Status + next step |

The optimized version (7 steps) merges the original steps 3–4 (Patente + Publikationen) into one "Prior Art suchen" step.

---

## Optimization Pipeline

The module uses an AI-driven optimization pipeline to improve the process:

### Phase 1: Generate
- **Inputs:** PROCESS-SCAFFOLD.md, USER-PERSONA.md, PROCESS-QUALITY-GRADE.md, source process (00–07)
- **Script:** `scripts/optimize_process.py`
- **Action:** GPT generates an optimized process variant that addresses quality feedback (e.g., combine steps, add risk warnings, clarify handoffs)
- **Output:** Raw Phase 1 response → parsed into `optimized/draft/`

### Phase 2: Review and Refine
- **Input:** Phase 1 output
- **Action:** GPT critically reviews Phase 1, applies improvements
- **Output:** Raw Phase 2 response → parsed into `optimized/refined/`

### Key Optimization Criteria
- Follow scaffold structure (Titel, Ziel, So gehst du vor, Tipp, Fazit, Nächster Schritt)
- Serve persona: simpler language, more concrete instructions, explicit input-output handoffs
- Add clear limitation/risk warning early
- Address quality-grade issues (e.g., Grade C feedback: too complex, vague tips)

## Processes Used to Optimize Results

1. **Scaffold-driven structure:** Every step follows a consistent pattern (Titel, Ziel, So gehst du vor, Tipp, Fazit, Nächster Schritt) to ensure clarity and navigability.

2. **Persona-based design:** All content is evaluated against USER-PERSONA.md—does it serve the "unsichere Erfinder:in mit konkreter Idee"?

3. **Quality grading:** grade_process.py produces a critical assessment (A–F) from multiple angles: persona fit, completeness, clarity, risk, usability, structure.

4. **Two-phase optimization:** Phase 1 generates improvements; Phase 2 reviews and refines, reducing logical gaps and redundancies.

5. **Example generation:** generate_examples.py creates coherent PROZESS-BEISPIEL.json showing how the process evolves step by step, used for validation and documentation.

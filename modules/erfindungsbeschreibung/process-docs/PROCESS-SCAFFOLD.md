## Mapping pro Schritt

### Schritt 0: Einleitung

**Zweck:** Warum gute Beschreibung wichtig ist. Typische Stolpersteine. Was dieser Guide macht.

| Element | Inhalt |
|---------|--------|
| Was dieser Guide macht | Kurze Erklärung; Thema: erklären vs. dokumentieren |
| Typische Stolpersteine | Zu viel/zu wenig; Warum vs. Was; Gleichungen |
| Wichtige Grenze | Keine Rechtsberatung; TTO-Formular als Ziel |
| Nächster Schritt | Link zu Schritt 1 |
| Besonderheiten | Kein Ziel, kein "So gehst du vor", kein Tipp. Rein konzeptuell. |

---

### Schritt 1: Kern der Neuerung identifizieren

**Zweck:** Was ist wirklich neu? Abgrenzung zu Vorarbeit.

| Element | Inhalt |
|---------|--------|
| Ziel | Kern in 1–2 Sätzen formulieren |
| So gehst du vor | Leitfragen; Abgrenzung zu Vorarbeit; Beispiel HM TI-RSS |
| Tipp | Typische Fehler (zu breit, zu technisch) |
| Fazit | Output: Kern in 1–2 Sätzen |
| Nächster Schritt | Link zu Schritt 2 |

---

### Schritt 2: Stand der Technik skizzieren

**Zweck:** Was existiert? Welche Nachteile haben bestehende Lösungen?

| Element | Inhalt |
|---------|--------|
| Ziel | Kurze Stichpunkte zu Prior Art und Nachteilen |
| So gehst du vor | Recherche; Lücken identifizieren; Beispiel HM Stand der Technik |
| Tipp | Nicht zu ausführlich; Fokus auf Nachteile |
| Fazit | Output: Stichpunkte Prior Art + Nachteile |
| Nächster Schritt | Link zu Schritt 3 |

---

### Schritt 3: Technische Lösung beschreiben

**Zweck:** Was muss man tun, damit es funktioniert? (nicht: warum es funktioniert)

| Element | Inhalt |
|---------|--------|
| Ziel | Klare Lösungsskizze |
| So gehst du vor | TUM-Regel: "what one must do so that it works"; Beispiel JMU |
| Tipp | Warum vs. Was; Gleichungen vermeiden |
| Fazit | Output: Lösungsskizze |
| Nächster Schritt | Link zu Schritt 4 |

---

### Schritt 4: Struktur und Form

**Zweck:** Technische Aufgabe → Stand der Technik → Lösung; 4 Seiten; Skizzen.

| Element | Inhalt |
|---------|--------|
| Ziel | Strukturierte Gliederung |
| So gehst du vor | HM-Gliederung als Vorlage; TUM Guideline (Ad 5.2); Umfang |
| Tipp | Skizzen einbinden |
| Fazit | Output: Gliederung |
| Nächster Schritt | Link zu Schritt 5 |

---

### Schritt 5: Typische Fehler vermeiden

**Zweck:** Zu viel/zu wenig; Gleichungen; Warum vs. Was.

| Element | Inhalt |
|---------|--------|
| Ziel | Checkliste vor Abgabe |
| So gehst du vor | Checkliste aus TUM, HM, JMU |
| Tipp | Durchschnittlich qualifizierte Fachperson als Zielgruppe |
| Fazit | Output: Checkliste |
| Nächster Schritt | Link zu Schritt 6 |

---

### Schritt 6: Ergebnis

**Zweck:** Zusammenfassung, nächste Schritte, TTO-Formular.

| Element | Inhalt |
|---------|--------|
| Ziel | Fertige Vorlage + nächste Aktion |
| So gehst du vor | Zusammenfassung; TTO-Formular finden; nächste Schritte |
| Fazit | Output-Checkliste |
| So geht's weiter | Tabelle (Option | Wann?) – Ausstiegsoptionen |
| Besonderheiten | Kein Tipp. "So geht's weiter" statt "Nächster Schritt". Prozess-Ende. |
